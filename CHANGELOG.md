# Changelog

## [1.1.4]

- **安全加固 (Security Hardening)**:
  - **DLL 加密 payload 参数大小限制**：在 `hq_crypto` 命令中，增加对 `payload` 参数 1MB 的上限大小校验，拦截超长参数，防范因命令行参数过长导致命令执行失败或内存溢出。
  - **二维码生成分辨率校验**：在 `generate_qr` 的 `resolution` 参数中，增加后端数值 clamp 逻辑（限制在 64 至 2048 之间），防止攻击者通过手动篡改客户端存储数据注入超大分辨率参数导致后端分配大内存发生 OOM 崩溃。
  - **Markdown 保存内容大小限制**：在 `save_md_file` 与 `save_md_file_as` 命令中，增加对 `content` 参数 10MB 的上限校验，拦截超长参数保存，防范大文件在 IPC 传递和磁盘写入时的性能及空间消耗问题。
  - **文件搜索查询字符串限制**：在 `search_files` 的 `query` 参数中，增加 1024 字符的长度上限校验，拦截超长查询输入，防范 CPU 和内存被过度消耗。

## [1.1.3]

- **局域网文件传输 (LAN File Transfer)**:
  - **历史记录重复与删除问题修复**：修复由于 React 严格模式导致的 Tauri 事件监听器重复注册的内存泄漏漏洞。优化历史记录保存与删除逻辑，在读写前拉取最新的 `localStorage` 真实内容并进行去重，防止多窗口实例并发更新导致的脏写覆盖。
  - **跨窗口状态实时同步**：添加对 `storage` 事件的监听，在任意窗口删除/清空历史记录时，其他所有运行中的窗口实例均会自动同步重载并刷新 UI。
  - **好友选择下拉框美化**：将对称加密的精美下拉框抽取为公共组件 `CustomSelect`，文件传输页面的好友选择框采用此样式，实现了圆角聚焦高亮、展开动画以及暗色系毛玻璃下拉菜单效果。
  - **已配对好友别名编辑**：在配对好友列表中新增就地修改别名的编辑铅笔图标，支持快捷输入框编辑，按 `Enter` 键或失去焦点 (`onBlur`) 自动调用 `update_peer_alias` 写入配置，按 `Escape` 键取消编辑，并同步增加了国际化翻译支持。
  - **发送文件交互流程优化**：优化配对好友列表中“发送文件”按钮点击无效/无反应的体验缺陷。现在点击该按钮后会自动调起系统的文件选择框，选定文件后自动展开右侧传输面板并将“接收对象”预设为当前点击的好友。
  - **输入框占位符视觉优化**：微调“添加对象”卡片中输入框占位符提示字颜色，保证在深浅色主题下都具有优秀的文字可读性与层次感。
  - **macOS 设备主机名获取修复**：修复 macOS 下通过读取 `HOSTNAME` 环境变量与 Linux 的 `/etc/hostname` 必定失败导致回退为通用设备名的 bug，改为优先调用标准的 POSIX `libc::gethostname` 系统 API 来提取真实的本机主机名（如 `xxx.local`）。
  - **界面自适应填充**：移除局域网文件传输页面外层容器的 `max-w-6xl` 最大宽度限制，使页面布局在窗口变大或最大化时能够自适应充满整个水平空间。
  - **定位文件功能修复**：修复 Windows 平台下，当接收文件保存路径中包含空格、括号等特殊字符时，点击文件夹图标无法正确在资源管理器中高亮选中文件，而是退回打开默认“文档”目录的 Bug。在 Rust 后端 `reveal_in_explorer` 命令中引入 `.raw_arg()` 精准包裹双引号并转换斜杠，避开 Rust 默认对含空格的整段参数包裹引号而导致 `explorer.exe` 识别失败的缺陷。
  - **接收失败前端卡死修复**：修复在断网、文件写入失败或发送端主动中断时，接收端后台任务报错退出却没有通知前端的 Bug。现在接收失败后会主动向前端发送 `recv-error` 事件，由前端更新传输状态，计入失败历史记录，并从“当前传输”列表中移除。
  - **拒收弹窗重复逻辑修复**：修复对方拒收时前端重复弹出“拒绝”与“发送失败”双重提示框的缺陷。在 Rust 层的发送任务抛出 `Rejected` 错误后，外层事件分发中增加了拦截逻辑，避免再次触发通用的 `send-error` 广播事件。
  - **发送端误报成功修复**：修复当发送文件提前结束或因读取异常中断（实际传输字节数小于文件总大小时），发送端仍错误标记传输成功并误写入成功历史记录的问题。增加了文件实际发送字节数与原文件大小一致性校验。
  - **配对设备身份校验安全加固**：增强了局域网受信任设备的校验安全性，从原先仅校验发送端 IP 的宽松逻辑，升级为 IP、监听端口（Port）以及设备主机名（Hostname）三重强关联验证，防止同一局域网下其他未配对的恶意设备/进程绕过安全审查窃取信任关系。同时保留了对旧版本客户端的平滑兼容。

- **安全加固 (Security Hardening)**:
  - **路径穿越漏洞防范**：在文件传输接收端增加 `safe_filename` 校验，拦截含有目录穿越（如 `../`）的恶意文件名，防止文件保存路径逃逸。
  - **防崩溃与异常处理**：替换 Rust 传输及搜索后端中多处 `.unwrap()` 调用（如 `search_files` 中对 `content_filter` 的读取），使用更稳健的异常处理与 `.expect()` 代替，防止异常情况下程序 panic 崩溃。
  - **系统路径校验**：在 `open_file` 与 `reveal_in_explorer` 命令执行系统调用前，增加本地文件或目录的存在性校验，规避无效路径的系统调用问题。
  - **文件删除路径沙箱校验**：在 `delete_local_file` 命令中，增加 `canonicalize` 路径规范化校验，强制待删除路径必须位于配置的 `save_dir` 或默认下载目录下，拒绝删除系统外部任意敏感文件。
  - **JAR/ZIP 包解包数量限制**：在 `list_jar_entries` 解析条目前增加 50,000 条条目的数量上限限制，拒绝加载解包包含过多条目的超大型压缩包（如 ZIP 炸弹），规避内存暴涨与界面冻结问题。
  - **好友请求等待超时处理**：在 `handle_incoming_connection` 中为好友请求响应通道 `rx` 增加 60 秒的超时逻辑，并在超时或异常断开后主动将请求从 `pending_friends` 列表中清除，杜绝因长时间无应答导致的 TCP 连接和 Tokio 协程任务无限挂起问题。
  - **搜索文件条数限制约束**：在 `search_files` 命令入口对前端传入的 `limit` 参数进行最大值校验（上限设为 2,000 条），防止通过 API 传入过大 limit 导致 SQLite 一次性返回海量数据而引发的内存暴涨隐患。
  - **XSS 安全防护**：修复 `EncoderDecoder` 在解码 HTML 实体时直接读写 `textarea.innerHTML` 的 XSS 潜在风险，升级为安全的浏览器 `DOMParser` 进行文本解析。
  - **接收大小上限限制**：在接收文件时增加 10GB 的文件大小上限校验，拒绝恶意大文件传输请求，保障本地磁盘安全。

- **界面交互与体验优化 (UX & Quality)**:
  - **全局 Toast 弹窗与内存清理**：将局域网文件传输页面的多处原生阻塞型 `alert()` 弹窗统一替换为非阻塞式、精美的页面浮动 Toast 提示组件，并在页面逻辑中消除了在 `setTransmissions` 中使用同名 `t` 变量导致的翻译函数变量遮蔽隐患；同时使用 `useRef` 及清理机制来清除 `setTimeout` 延迟定时器，消除组件卸载时的内存泄漏隐患。
  - **好友请求静默覆盖拒绝**：优化前端 `friend-request` 事件监听，在新好友请求到达并覆盖旧请求前，自动调用 `respond_friend_request` 接口向后端发送拒绝响应，释放后端旧请求的挂起资源。
  - **文件传输对端 IP 追踪**：修复文件传输历史记录中 `peerIp` 字段为空的 Bug，保证接收与发送时的对端 IP 能正确记录到历史记录中。
  - **代码对比拖拽功能增强**：在 `FileDiff` 拖拽区域支持真实文件拖放读取（限制大小为 10MB），并在读取前校验文件名后缀，禁止强行读取非文本二进制文件（如 `.zip`，`.exe`）引发的乱码问题。
  - **加解密输入转换提示**：修复 `CryptoTool` 在 RSA/SM2 加密中 HEX/BASE64 格式输入明文解析失败时静默忽略并强行对乱码加密的问题，改为直接抛出“HEX/BASE64 输入格式不合法”的错误给用户提示。
  - **路由逻辑重构**：重构 `App.tsx` 中的路由渲染条件判断，提取 `isToolActive` 辅助函数，降低条件表达式冗余度。
  - **硬编码报错翻译**：将 `hq_crypto` 中硬编码的中文报错信息（如“执行超时”及“HQ 库执行错误”）重构翻译为英文，使后端报错格式更加规范。

## [1.0.9]

- **加解密工具 (CryptoTool)**:
  - **3DES 16 字节密钥处理优化**：当输入密钥正好是 16 字节（128位）时，采用标准的 2-key 3DES 逻辑使 $K_3 = K_1$ 生成合法的 24 字节密钥进行加解密，与行业标准工具完全兼容，且不再触发长度警告。
  - **非阻塞 RSA 4096 密钥生成**：引入 Web Crypto API 异步生成强密钥对，在后台线程处理生成计算，彻底避免同步计算占用主 JS 线程导致应用界面和按钮动画卡死的问题。
  - **密钥生成错误捕获与 UI 提示**：优化了 RSA 和 SM2 密钥生成异常流程，生成失败时错误消息能实时显示到界面上，提升自愈感知能力。
  - **CryptoJS 严格类型安全化**：移除了 `any` 类型的配置定义，直接映射 `typeof CryptoJS.mode.CBC` 等严格类/接口类型，保障加解密配置 `cfg` 的绝对类型安全。
  - **获取警告性能优化**：将 `getKeyIvWarnings` 逻辑封装于 `useMemo`，极大降低在配置项改变或按键交互时的无效计算开销。
- **系统核心与性能优化 (Core & Performance)**:
  - **首屏路由懒加载 (Code Splitting)**：全面将主页面各工具组件调整为 React `lazy` + `Suspense` 动态引入。首次加载的主 JS 包大小由 **1.58 MB** 骤降至 **238 KB**，消除了 Vite 打包提示，并大幅加快了 Tauri 应用的冷启动渲染速度。
  - **懒加载组件错误边界防护 (ErrorBoundary)**：添加自定义 React `ErrorBoundary` 错误边界组件，若本地加载代码 Chunk 异常时会自动渲染友好的防白屏恢复页，并支持一键刷新，提升客户端在极地状况下的容错性。
  - **i18n 编译兼容性修复**：支持包含变量的 `t(key, params)` 替换模板，并使用 `.split().join()` 实现替换逻辑，彻底解决低 ES 编译目标下因 `replaceAll` 导致的 TypeScript 编译报错。
