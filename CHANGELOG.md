# Changelog

## [1.0.9]

- **加解密工具 (CryptoTool)**:
  - **3DES 16 字节密钥处理优化**：当输入密钥正好是 16 字节（128位）时，采用标准的 2-key 3DES 逻辑使 $K_3 = K_1$ 生成合法的 24 字节密钥进行加解密，与行业标准工具完全兼容，且不再触发长度警告。
  - **非阻塞 RSA 4096 密钥生成**：引入 Web Crypto API 异步生成强密钥对，在后台线程处理生成计算，彻底避免同步计算占用主 JS 线程导致应用界面和按钮动画卡死的问题。
  - **密钥生成错误捕获与 UI 提示**：优化了 RSA 和 SM2 密钥生成异常流程，生成失败时错误消息能实时显示到界面上，提升自愈感知能力。
  - **CryptoJS 严格类型安全化**：移除了 `any` 类型的配置定义，直接映射 `typeof CryptoJS.mode.CBC` 等严格类/接口类型，保障加解密配置 `cfg` 的绝对类型安全。
  - **获取警告性能优化**：将 `getKeyIvWarnings` 逻辑封装于 `useMemo`，极大降低在配置项改变或按键交互时的无效计算开销。
- **系统核心与性能优化 (Core & Performance)**:
  - **首屏路由懒加载 (Code Splitting)**：全面将主页面各工具组件调整为 React `lazy` + `Suspense` 动态引入。首次加载的主 JS 包大小由 **1.58 MB** 骤降至 **238 KB**，消除了 Vite 打包提示，并大幅加快了 Tauri 应用的冷启动渲染速度。
  - **懒加载组件错误边界防护 (ErrorBoundary)**：添加自定义 React `ErrorBoundary` 错误边界组件，若本地加载代码 Chunk 异常时会自动渲染友好的防白屏恢复页，并支持一键刷新，提升客户端在极地状况下的容错性。
  - **i18n 编译兼容性修复**：支持包含变量的 `t(key, params)` 替换模板，并使用 `.split().join()` 实现替换逻辑，彻底解决低 ES 编译目标下因 `replaceAll` 导致的 TypeScript 编译报错。

## [1.0.8]

- **系统核心 (Core)**:
  - 生产环境 CSP 兼容性修复：将内联主题初始化脚本提取为独立文件 `public/theme-init.js`，彻底解决生产构建下的 FOUC（页面闪烁）问题。
- **文本转二维码 (TextToQr)**:
  - 修复「清空」按钮提示文字的硬编码英文问题，接入 i18n 国际化。
- **文本对比 (FileDiff)**:
  - 移除 Tauri 2 环境下不再执行的浏览器原生 `FileReader` 拖放死代码，仅保留纯文本拖放逻辑，精简代码结构。
- **JAR 查看器 (JarViewer)**:
  - 彻底解决 Windows 虚拟机网络映射盘 (Z:) 或 UNC 路径下执行反编译时抛出 `ClassNotFoundException: org.benf.cfr.reader.Main` 的错误。现无条件将 CFR 工具包复制到本地 C 盘临时目录执行，消除网络盘类加载器权限障碍。
  - 新增临时文件夹防清理自愈机制：如果应用运行期间操作系统的临时文件夹被第三方清理软件或用户手动清空，应用会在下次反编译前自动检测丢失并从自带资源无缝拷贝恢复，保障反编译服务永不中断。

- **JAR 查看器 (JarViewer)**:

## [1.0.3]

- **JAR 查看器 (JarViewer)**：

- **文档同步**：
  - 在 `AGENTS.md` 中补充记录 JAR Viewer 模块及其四个 Tauri 命令。
